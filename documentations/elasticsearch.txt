Si deseas quitar la contraseña de Elasticsearch y desactivar la autenticación, puedes hacerlo deshabilitando la seguridad en su archivo de configuración. Aquí te dejo los pasos para hacerlo:

Pasos para deshabilitar la autenticación en Elasticsearch:
Accede al contenedor de Elasticsearch (si estás usando Docker):

bash
Copiar
Editar
docker exec -it elasticsearch-server bash
Edita el archivo de configuración de Elasticsearch: El archivo de configuración se encuentra generalmente en /usr/share/elasticsearch/config/elasticsearch.yml.

Puedes editarlo usando un editor de texto, como nano o vi. Por ejemplo:

bash
Copiar
Editar
nano /usr/share/elasticsearch/config/elasticsearch.yml
Desactiva la seguridad de Elasticsearch: Dentro de este archivo, busca las siguientes líneas (o agrégalas si no existen) y configúralas para desactivar la seguridad:

yaml
Copiar
Editar
xpack.security.enabled: false
xpack.security.transport.ssl.enabled: false
Reinicia Elasticsearch: Después de guardar los cambios, reinicia Elasticsearch para que se apliquen las configuraciones:

bash
Copiar
Editar

O si estás ejecutando Elasticsearch directamente en tu máquina sin Docker, simplemente reinicia el servicio de Elasticsearch:

bash
Copiar

Una vez que Elasticsearch se haya reiniciado, intenta hacer una solicitud curl sin autenticación:

-------------------COMANDOS DOCKER-------------

Opción 1: Detener todos los contenedores en ejecución

docker stop $(docker ps -q)
docker ps -q: obtiene solo los IDs de los contenedores en ejecución.

docker stop: los detiene.

🧨 Opción 2: Detener y eliminar todos los contenedores (¡con cuidado!)

docker rm -f $(docker ps -aq)
-f: fuerza el stop antes de eliminar.

-a: incluye contenedores parados también.

⚠️ Esto elimina los contenedores, pero no borra los volúmenes ni las imágenes.

🧘 Opción 3: Detener contenedores específicos
Por ejemplo:

docker stop qdrant-server minio-server redis-server elasticsearch-server
🧼 ¿Y si quiero también eliminar los contenedores (pero no los datos)?

docker rm qdrant-server minio-server redis-server elasticsearch-server
Como ya usaste --restart unless-stopped, los contenedores no se volverán a iniciar automáticamente después de ser eliminados.

Si usaste volúmenes (como -v qdrant_data:/qdrant/storage), los datos se conservarán.
